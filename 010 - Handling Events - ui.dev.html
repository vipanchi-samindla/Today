Handling Events

To build a modern, interactive application, you’re going to need the ability to listen to and respond to events – obviously. Thankfully, the API to do this in React is fairly simple and it involves two things you’re already familiar with – props and functions.

Let's start off with the most basic example, alerting OUCH whenever a user clicks on a button.

First, we'll start with a button element.

export default function AlertButton() {
return (
<button>
Alert
</button>
)
}

Now let's create a function which will encapsulate the logic for what happens when a user clicks on the button. We'll call it handleClick.

export default function AlertButton() {
  const handleClick = () => alert("OUCH")

  return (
    <button>
      Alert
    </button>
  )
}

Finally, we want to tell React to invoke our handleClick function whenever a user clicks on our button element. To do that, we can pass button an onClick prop, which will be set to our handleClick function.

export default function AlertButton() {
  const handleClick = () => alert("OUCH")

  return (
    <button onClick={handleClick}>
      Alert
    </button>
  )
}

Now, whenever our button is clicked, React will invoke handleClick.

Basic, yes – but this example demonstrates some important concepts and patterns that are consistent regardless of the complexity of the event you're listening for.

First, it's a good idea to encapsulate the event handler into its own function, and it's common practice to name the function handle + the event name – in our case handleClick.

const handleClick = () => {}


const handleFocus = () => {}


const handleChange = () => {}

Next, you'll notice that we're passing our event handler as a reference to onClick, not as an invocation. Note the difference between these two onClick props.

<button onClick={handleClick}>
  Passing a Reference
</button>


<button onClick={handleClick()}>
  Passing an Invocation
</button>

For the record, this is true for passing any function as a prop and not just event handlers.

If we pass onClick an invocation, à la handleClick(), as soon as the JavaScript engine encounters the invocation, it will invoke the function and pass the return value to onClick. This is obviously not what we want.

Instead, we want to pass onClick a reference to the function, so that it can invoke the function when the event occurs.

Next, you'll notice that we created our event handler inside the component, not outside of it. This is also a common pattern, but it may feel unintuitive at first. Can you think of why we do this?

To see why, let's adjust our example a bit. Instead of alerting OUCH, let's alert whatever is passed in as a message prop to our AlertButton component.

export default function AlertButton({ message }) {
  const handleClick = () => alert(message)


  return (
    <button onClick={handleClick}>
      Alert
    </button>
  )
}

By declaring our handleClick function inside of our component, we automatically have access to the message prop via JavaScript's normal lexical scoping rules.

Notice what happens when we move handleClick outside of the component.

const handleClick = (msg) => alert(msg)

export default function AlertButton({ message }) {
  return (
    <button onClick={() => handleClick(message)}>
      Alert
    </button>
  )
}

We no longer have access to message via scope. Instead, we have to pass it as an argument to handleClick, which forces us to create another function that we pass to onClick. This indirection is unnecessary and is a good indication that we should move handleClick back inside of our component so we can take advantage of lexical scoping.

export default function AlertButton({ message }) {
  const handleClick = () => alert(message)

  return (
    <button onClick={handleClick}>
      Alert
    </button>
  )
}

One reason why your intuition might be telling you to move handleClick outside of the component is because you're worried about performance. After all, if it's inside the component, then JavaScript will have to create a new function every time the component renders, right?

Correct, and that's (mostly) fine.

Turns out, JavaScript is really good at creating and destroying functions. To the point where it's usually a non-issue. For the very few edge cases where this isn't true, we have some options we'll learn about later in the course.

So at this point you know how to add a click handler to a button (which is cool), but let's look at a scenario that isn't straight out of a generic React tutorial.

Say we wanted to build something like this, a little game where the user has a certain amount of time to copy the main emoji from a list of smaller emojis below.

Try it for yourself. Hit RESET then copy (using your keyboard or mouse) whatever the highlighted emoji is.

Fun. Let's start off with the easy parts, just the UI.

We'll have an array of emojis, a function for selecting a random emoji, and some JSX.

import emojis from "./emojis"

const selectRandomElement = (arr) => {
  return arr[Math.floor(Math.random() * arr.length)]
}

export default function App () {
  const selected = selectRandomElement(emojis)

  return (
    <div>
      <h1>{selected.emoji}</h1>
      <p>copy this emoji</p>
      <ul>
        {emojis.map(({ id, emoji}) => {
          return (
            <li key={id}>
              {emoji}
            </li>
          )
        })}
      </ul>
      <button>Reset</button>
    </div>
  )
}

Now let's work on our event handler that we'll pass as an prop to each of our three emojis.

This event handler will need to check both if the user copied the right emoji, as well as if they did it in under 5 seconds. If both of those conditions are true, we'll alert the user that they won. If not, we'll tell them why they lost.

This does bring up a new scenario though. We now need to know some specific information about the event that was triggered. Specifically, we need to know which emoji was copied so we can see if it was the right one.

This brings us to the last piece of knowledge you need for managing events in React. Whenever an event handler is called by React, React will pass an object as the first argument to the event handler that contains all the information about the event that was triggered.

In our example, that event object will look something like this.

SyntheticBaseEvent {
  bubbles: true,
  cancelable: true,
  clipboardData: DataTransfer {},
  currentTarget: null,
  defaultPrevented: false,
  eventPhase: 3,
  target: li,
  timeStamp: 52607.59999999404,
  type: "copy",
}

Though there's a lot of extraneous data here, there are a few things that'll be helpful to us – namely the target and timeStamp properties.

target is the element that triggered the event (which we'll be able to use to figure out which emoji was copied) and timeStamp tells us how long it's been since the document finished loading and the event occurred (which we'll be able to use to figure out if the user was fast enough).

With that knowledge, building out our event handler is trivial. We'll compare e.target.innerText to the selected emoji to see if the user selected the right one and we'll check if e.timeStamp is less than 5000 to see if the user selected the emoji in time.

import emojis from "./emojis"

const selectRandomElement = (arr) => {
  return arr[Math.floor(Math.random() * arr.length)]
}

export default function App () {
  const selected = selectRandomElement(emojis)

  const handleCopy = (e) => {
    if (e.target.innerText !== selected.emoji) {
      alert("Wrong emoji")
    } else if (e.timeStamp > 5000) {
      alert("Too slow. Reset and try again.")
    } else {
      alert("🎉 Winner!")
    }
  }

  return (
    <div>
      <h1>{selected.emoji}</h1>
      <p>copy this emoji</p>
      <ul>
        {emojis.map(({ id, emoji}) => {
          return (
            <li onCopy={handleCopy} key={id}>
              {emoji}
            </li>
          )
        })}
      </ul>
      <button>Reset</button>
    </div>
  )
}

Now the only thing left to do is create our event handler for our reset button.

This one is simple, we'll just call location.reload() to reload the page.

import emojis from "./emojis"

const selectRandomElement = (arr) => {
  return arr[Math.floor(Math.random() * arr.length)]
}

export default function App () {
  const selected = selectRandomElement(emojis)

  const handleCopy = (e) => {
    if (e.target.innerText !== selected.emoji) {
      alert("Wrong emoji")
    } else if (e.timeStamp > 5000) {
      alert("Too slow. Reset and try again.")
    } else {
      alert("🎉 Winner!")
    }
  }

  const handleReset = () => {
    window.location.reload()
  }

  return (
    <div>
      <h1>{selected.emoji}</h1>
      <p>copy this emoji</p>
      <ul>
        {emojis.map(({ id, emoji}) => {
          return (
            <li onCopy={handleCopy} key={id}>
              {emoji}
            </li>
          )
        })}
      </ul>
      <button onClick={handleReset}>Reset</button>
    </div>
  )
}

And that's it. Now, if the user copies the right emoji in under 5 seconds, they'll get a winner alert.
